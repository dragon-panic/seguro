## Observed

`seguro/src/config.rs::runtime_dir()` returns `$XDG_RUNTIME_DIR/seguro` (tmpfs on systemd, typically 10% of RAM, capped ~6 GB). Every per-session artifact lives there: vhost-user sockets, `qemu.pid`, `ssh.port`, `cidata.img`, `session.json`, SSH host key, and — critically — `session.qcow2`.

During a 2-runner ox pool build in `~/projects/elu` (2026-04-17 16:49 UTC), the two session.qcow2 files grew to 4.5 GB + 1.8 GB = 6.3 GB, filling `/run/user/1000` (100%). Guest writes started failing, the ox-runner heartbeat loop wedged, and both VMs silenced at 0 % CPU within 2 seconds of each other. The herder's `heartbeat_missed` → `re-dispatching orphaned step` logic could not progress because the pool's runner slots were still marked executing.

## Root cause

Two artifact classes with different lifetime/size profiles share one root:

- **Sockets / pids / metadata**: ephemeral, small (KB), want tmpfs — fast IPC, auto-cleared on reboot.
- **qcow2 overlay**: grows with guest work, multi-GB during a build, wants real disk.

`runtime_dir()` correctly picks tmpfs for the first class; the second class is along for the ride.

## Approach (two slices)

### Slice 1 — storage split (this node's children will cover it)
- New `overlay_dir()` in `config.rs` → `$XDG_STATE_HOME/seguro/overlays` by default; `SEGURO_OVERLAY_DIR` env override.
- `Session::allocate` builds `overlay_path` from `overlay_dir()`. Creation order: runtime_dir first, then overlay.
- `Session::cleanup` deletes the overlay file explicitly in addition to the runtime dir.
- `classify_sessions`: "overlay file missing" ⇒ Dead.

### Slice 2 — orphan sweep in prune
- `disk_ids − live_ids` pass with mtime grace. Covers reboot fallout (tmpfs gone, overlays survive).

## Out of scope

Making `--persistent` survive reboot — requires moving `session.json` to disk as well. Document the current limitation; file a follow-up if wanted.

3. Watch `df -h /run/user/\$UID` trend to 100 %. Writes inside the guest start hanging.
