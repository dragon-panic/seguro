## Problem

Seguro is coupled to claude-code: credential injection, persona CLAUDE.md,
AI-provider-aware proxy (token counting, Anthropic header parsing). The
credential `rm` on ro mounts causes SSH exit 255.

Seguro should be a dumb VM runtime with a dumb proxy.

## Changes

### Strip agent-specific concerns
1. **`inject_workspace_config()`** — remove entirely (no `.seguro/` in shares)
2. **Credential/env-sourcing in `build_mount_preamble()`** — keep only mount + cd
3. **`PersonaConfig`** + `load_persona()` + `persona_config` field
4. **API key passthrough** in run.rs (ANTHROPIC_API_KEY, OPENAI_API_KEY, GITHUB_TOKEN)

### Strip AI-awareness from proxy (keep proxy itself)
5. **`src/proxy/ai_usage.rs`** — delete entirely (Provider, ProviderMap, token parsing)
6. **AI counters from ProxyStats** — remove ai_requests, ai_input_tokens, etc.
7. **`commands/api_usage.rs`** — delete (the `seguro api-usage` subcommand)
8. **Provider config in Config** — remove `[providers]` section

### Keep in proxy
- HTTP/HTTPS CONNECT forwarding
- Host allow/deny filtering (filter.rs)
- Request logging (log.rs) — just URL + status, no body inspection
- TLS inspection (ca.rs) — for full URL logging if desired
- All net modes including api-only (proxy enforces allow-list)
